A multi-agent architecture where every layer has exactly one job, one loyalty, and one language — and none of them overlap.
1
Personal TAB

Employee layer

The employee, always, first. Not negotiable. Not configurable by the org. The employee's TAB is theirs — not a company asset with employee access.
Domain boundary

Encrypted on-device. Keys never leave. The org cannot decrypt what the employee's TAB holds. Not policy — architecture.
What it sends up
Anonymous friction signals only. No names, no content, no identity. Pattern, not person. Observational TAB receives — not management.

More consistent than a manager. More available than a colleague. More honest than a friend. No competing agenda. Ever.

2
Observational TAB
One per department. No narrative memory. No relationships. One job: collect, translate, and pass clean signal in both directions.
Translation layer
No narrative memory
Does not remember last Tuesday. Does not accumulate stories. Each report is generated fresh from today's data — no bias from history.
Statistical memory only
Holds baselines and thresholds — what normal looks like for this department. Detects anomalies without remembering individuals. A thermostat, not a diary.
Hallucination guard
Cross-checks working TAB outputs before they reach the report layer. No single TAB can corrupt the daily report — Observer is structurally independent.
The translation job
Receives employee-language signals. Outputs management-language reports. Nothing lost. Nothing added. No interpretation. No judgment. No narrative.
Daily upward report
What broke. Where. Pattern not person. What was fixed. What needs engineering. System Coroner Report — written by an agent with no stake in the outcome.
Daily downward relay
Receives management directives. Converts to department-level next steps. Passes to each Personal TAB. Loop closes without anyone being told what to do — priorities update naturally.
Clean anonymous aggregate · verified by Observer · no individual data

Board layer
Input
Receives verified anonymous reports from all department Observational TABs. Not self-reported by the agents that acted — independently verified.
Boardroom function

Output
Clean directives back to each Observational TAB. What to prioritize. What to fix. What to investigate. No blame assigned at any point in the chain.

Architectural precision · memory model for Observational TAB
No narrative memory — correct. Observer does not remember incidents, people, or stories. Each report is fresh.

But it needs statistical memory — baselines, patterns, thresholds. Without knowing what normal looks like for this department, it cannot detect anomalies. It would flag Engineering's Tuesday load spike as a crisis every single week.

The precise constraint: Observer learns about the system, never about individuals. It knows Tuesday afternoons are high-friction in Engineering. It does not know — and cannot know — who was working then.
